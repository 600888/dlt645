class src.protocol.protocol.DLT645Protocol[源代码]

基类：object

DLT645协议实现类

该类实现了DLT645协议的核心功能，包括数据编码/解码、校验和计算、 帧构建和解析等操作。所有方法均为类方法，可直接调用。

classmethod decode_data(data)[源代码]

数据域解码（±33H转换）

按照DLT645协议规定，对数据域进行解码处理， 每个字节减去0x33H，并使用模256运算确保结果在0-255范围内。

返回类型:

bytes

Args:

data: 需要解码的数据字节

Returns:

bytes: 解码后的原始数据

classmethod calculate_checksum(data)[源代码]

校验和计算（模256求和）

按照DLT645协议规定，计算数据的校验和， 对所有字节进行累加，然后对结果取模256。

返回类型:

int

Args:

data: 需要计算校验和的数据字节

Returns:

int: 计算得到的校验和值（0-255范围）

classmethod encode_data(data)[源代码]

数据域编码（+33H转换）

按照DLT645协议规定，对数据域进行编码处理， 每个字节加上0x33H，并使用模256运算确保结果在0-255范围内。

返回类型:

bytes

Args:

data: 需要编码的原始数据字节

Returns:

bytes: 编码后的数据

classmethod build_frame(addr, ctrl_code, data)[源代码]

帧构建（支持广播和单播）

按照DLT645协议规定，构建完整的协议帧，包括前导字节、 地址域、控制码、数据域、校验和和结束标志。

返回类型:

bytearray

Args:

addr: 设备地址，6字节 ctrl_code: 控制码 data: 原始数据域

Returns:

bytearray: 完整的DLT645协议帧

Raises:

ValueError: 当地址长度不为6字节时抛出

classmethod deserialize(raw)[源代码]

将字节切片反序列化为 Frame 结构体

从原始字节数据中解析出DLT645协议帧。如果无法找到完整的帧， 则抛出异常。

返回类型:

Optional[Frame]

Args:

raw: 输入的原始字节数据

Returns:

Optional[Frame]: 解析出的Frame对象，如果解析失败返回None

Raises:

Exception: 当无法找到完整帧时抛出

classmethod deserialize_with_remaining(raw)[源代码]

将字节切片反序列化为 Frame 结构体，并返回未解析的剩余数据

从原始字节数据中解析出DLT645协议帧，并返回未解析的剩余数据。 如果数据不完整或无法解析，返回(原始数据, None)。

tuple[bytes, Optional[Frame]]

Returns:

tuple[bytes, Optional[Frame]]: (未解析的剩余数据, 解析出的帧) 如果数据不完整或无法解析，返回(原始数据, None)
